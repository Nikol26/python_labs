# Лабораторная работа №10  
## Структуры данных: Stack, Queue, Linked List

### Цель работы
Изучить базовые структуры данных: стек, очередь и односвязный список,
а также проанализировать их асимптотическую сложность.

---

## Стек (Stack)
Стек  структура данных, работающая по принципу LIFO (Last In, First Out).

### Основные операции:
- push  добавление элемента (O(1))
- pop  удаление верхнего элемента (O(1))
- peek  просмотр верхнего элемента (O(1))

Реализация выполнена на базе встроенного списка Python.

---

## Очередь (Queue)
Очередь  структура данных, работающая по принципу FIFO (First In, First Out).

### Основные операции:
- enqueue  добавление элемента (O(1))
- dequeue  удаление первого элемента (O(1))
- peek  просмотр первого элемента (O(1))

Реализация выполнена с использованием collections.deque.

---

## Односвязный список (SinglyLinkedList)
Односвязный список состоит из узлов, каждый из которых хранит значение
и ссылку на следующий узел.

### Основные операции:
- append  добавление в конец (O(1))
- prepend  добавление в начало (O(1))
- insert  вставка по индексу (O(n))
- remove_at  удаление по индексу (O(n))

---

## Выводы
Стек и очередь обеспечивают быстрый доступ к элементам за счёт использования
оптимальных встроенных структур данных Python.  
Односвязный список менее эффективен при доступе по индексу, однако удобен
для динамических вставок и удалений элементов.
